# LuminaDB Backend Orchestrator

This folder contains the core **Express / Node.js** bridge that connects the UI, the Natural Language service, and the MongoDB cluster.

## Features
- **Mongoose Models**: Well-defined schemas for a Retail environment including `User`, `Product`, and `Order`.
- **API Routing**: A single `/api/query` POST endpoint that acts as the grand orchestrator.
- **Dynamic Execution**: It securely maps the incoming JSON generated by the NLP service (e.g., `{ collection: "Product", action:"find" }`) and executes the corresponding native Mongoose promises seamlessly.

## Getting Started

1. Navigate to this directory:
   ```bash
   cd LuminaDB/backend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Set your Environment Variables:
   - Create a file named `.env`.
   - Add your connection string, e.g., `MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/luminadb`.
4. Run the Express server:
   ```bash
   node index.js
   ```

The server will be listening at `http://localhost:5000`.

## Seeding Sample Data
If your MongoDB database is entirely empty, you can populate it immediately to test the web app context. 

While the backend server is stopped, simply run:
```bash
node seed.js
```
This will insert dummy users, high-end electronics, furniture products, and assorted test orders.
